# Shell & Terminal Guide

> 中文导读：本文介绍这套 dotfiles 的 shell（Zsh + Powerlevel10k）和两个终端模拟器（Wave Terminal、iTerm2）是如何配置的，以及怎么用。两个终端共用同一套 **Tokyo Night** 配色，所以无论你打开哪个，外观都是一致的。Wave 还提供了一组 `wave` / `wavep` / `waveweb` 函数，把当前标签页布置成类似 VS Code 的「编码助手 IDE」布局。

This repo configures your interactive shell and both terminal apps. Everything here is installed by `install.sh`:

| Piece | Source in repo | Installed to |
| --- | --- | --- |
| Zsh config | `shell/zshrc` | `~/.zshrc` (symlink) |
| Powerlevel10k prompt | `shell/p10k.zsh` | `~/.p10k.zsh` (symlink) |
| Powerlevel10k engine | cloned by installer | `~/powerlevel10k/` |
| Wave settings | `terminal/waveterm/settings.json` | `~/.config/waveterm/settings.json` |
| Wave terminal theme | `terminal/waveterm/termthemes.json` | `~/.config/waveterm/termthemes.json` |
| iTerm2 profile | `terminal/iterm2/DynamicProfiles/tokyo-night.json` | `~/Library/Application Support/iTerm2/DynamicProfiles/tokyo-night.json` |
| Nerd Font + Menlo | Homebrew cask | `~/Library/Fonts/…` |

Wave + iTerm2 are part of the **`full`** profile only. The **`claude`** profile installs just the shell pieces (Zsh, Powerlevel10k, fonts, the Claude CLI).

## 1. Zsh (`shell/zshrc` → `~/.zshrc`)

中文：`~/.zshrc` 是软链接到仓库里的 `shell/zshrc`，所以你改仓库里的文件就等于改了 shell 配置。下面逐段解释。

### Powerlevel10k prompt

The prompt is [Powerlevel10k](https://github.com/romkatv/powerlevel10k). Two parts make it work:

1. **The engine** is git-cloned to `~/powerlevel10k/` by the installer (`ensure_p10k`):

   ```bash
   git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ~/powerlevel10k
   ```

   `.zshrc` sources the theme from there:

   ```zsh
   source ~/powerlevel10k/powerlevel10k.zsh-theme
   ```

2. **The config** lives at `~/.p10k.zsh` (symlinked from `shell/p10k.zsh`). It was generated by the `p10k configure` wizard (powerline / rainbow / 2-line / 12h time / transient prompt). To regenerate it, run `p10k configure` and the wizard overwrites `~/.p10k.zsh`.

**Instant prompt.** The very top of `.zshrc` enables Powerlevel10k *instant prompt* so the prompt appears immediately on shell startup, before the rest of the rc finishes loading:

```zsh
typeset -g POWERLEVEL9K_INSTANT_PROMPT=quiet
if [[ -r "${XDG_CACHE_HOME:-$HOME/.cache}/p10k-instant-prompt-${(%):-%n}.zsh" ]]; then
  source "${XDG_CACHE_HOME:-$HOME/.cache}/p10k-instant-prompt-${(%):-%n}.zsh"
fi
```

> Important: any command that prints to the console or asks for input (passwords, `[y/n]` prompts) must go **above** this block. Everything else can go below. This is why the file is ordered the way it is.

### PATH additions

`.zshrc` prepends several directories to `PATH`, in this order (later `export`s end up earlier on `PATH`):

| Directory | Why |
| --- | --- |
| `~/.local/bin` | user-installed CLIs (pip `--user`, the Claude Code CLI installer drops here, etc.) |
| `~/bin` | this repo's scripts — `install.sh` symlinks everything in `bin/` here |
| `~/.bun/bin` | the [Bun](https://bun.sh) JS runtime / package manager |
| `~/Library/Application Support/waveterm/bin` | Wave's `wsh` helper CLI (so it works in plain login shells, not just inside Wave panes) |

```zsh
export PATH="$HOME/.local/bin:$PATH"
export PATH="$HOME/bin:$PATH"

# bun
export BUN_INSTALL="$HOME/.bun"
export PATH="$BUN_INSTALL/bin:$PATH"

# wsh, even outside Wave
export PATH="$HOME/Library/Application Support/waveterm/bin:$PATH"
```

### Bun completions

If Bun's completion file exists it is sourced so you get tab-completion for `bun`:

```zsh
[ -s "$HOME/.bun/_bun" ] && source "$HOME/.bun/_bun"
```

### Small aliases

```zsh
alias cl='clear'
alias check='history 0 | grep'   # search your shell history: `check git rebase`
```

### The `~/.zshrc.local` escape hatch

中文：所有「这台机器专属」或「带密钥」的配置都不应该提交到仓库。把它们放进 `~/.zshrc.local`（不被 git 跟踪），`.zshrc` 最后一行会自动加载它。

The last line of `.zshrc` sources a machine-local, **untracked** file if it exists:

```zsh
[[ -f ~/.zshrc.local ]] && source ~/.zshrc.local
```

Put anything secret or per-machine there: API keys, proxy settings, work-only aliases, `PATH` tweaks for this Mac only. It never gets committed, so the public repo stays clean.

### The deliberately-omitted `claude --dangerously-skip-permissions` alias

中文：源机器上把 `claude` 别名成了「永远跳过权限确认」。这是个危险习惯，本仓库**故意不带**这个别名——在新机器上自动跳过权限确认很容易出事。如果你确实想要，自己在 `~/.zshrc.local` 里加。

The original machine aliased `claude` to always pass `--dangerously-skip-permissions`. That auto-approves every tool/permission prompt, which is a footgun on a fresh machine. This repo intentionally does **not** ship it. It is left in `.zshrc` only as a commented opt-in:

```zsh
# alias claude='claude --dangerously-skip-permissions'
```

If you genuinely want it, add that line yourself in `~/.zshrc.local` (your untracked file), not in the tracked config.

## 2. The Wave functions

中文：这一组函数依赖 Wave 的命令行工具 `wsh`，把当前标签页的窗格布局成「左 shell / 中 Claude+Codex / 右 文件树」的编码助手 IDE。它们只在 Wave 终端里有意义。

These helpers all drive Wave through its `wsh` CLI. They only work **inside a Wave pane** (they read `$WAVETERM_TABID` / `$WAVETERM_BLOCKID`).

### `wave [dir]` — VS Code-like layout

Think of it like `code .`: it lays out the **current Wave tab** into a coding-agent IDE.

| Pane | Contents |
| --- | --- |
| Left | the shell you ran `wave` in (your session slot) |
| Middle | a `claude` agent **and** a `codex` agent, both `cwd`-ed into the target dir |
| Right | a file-tree view of the target dir |
| Bottom (optional) | a status/quota pane — only if you set `$WAVE_STATUS_CMD` (see below) |

Usage:

```bash
wave            # lay out $PWD
wave .          # same
wave ~/proj     # lay out a specific directory

WAVE_AI_PANEL=1 wave .   # also open Wave's built-in AI side panel
```

How it works under the hood: it diffs `wsh blocks list` before/after each `wsh run` to discover the new block IDs, then `wsh focusblock`s back to your shell. The agents are launched with `wsh run --cwd "$dir" -c 'claude'` and `… -c 'codex'`; the tree is `wsh view "$dir"`.

Two important caveats baked into the function's own notes:

- **Open the tab first.** Wave (tested on v0.14.5) can't create a tab from the CLI, so press `Cmd+T` for a fresh tab, then run `wave` inside it.
- **Resize by hand.** Wave can't script exact pane sizes from the CLI, so after layout you drag the dividers to size the panes. The function prints the new block IDs and a reminder when it finishes.

**Optional bottom status pane.** The original setup had a quota status bar; that script (`wave-status.sh`) is **not** part of this package. If you have your own status command, point `WAVE_STATUS_CMD` at it and `wave` will add a bottom pane running it:

```bash
export WAVE_STATUS_CMD="$HOME/bin/my-status.sh"   # e.g. in ~/.zshrc.local
wave .
```

### `wavep [-m] <path|url>` — preview block

Opens a file, image, directory, or URL in a Wave **preview** block (via `wsh view`). Local paths are resolved to absolute, symlink-resolved paths. `-m` / `--magnified` opens it magnified.

```bash
wavep report.pdf
wavep ./screenshots/
wavep -m diagram.png
```

### `waveweb [-m] <file.html|url>` — web browser block

Opens an HTML artifact, a `localhost` URL, or any URL in a Wave **web browser** block (via `wsh web open`). Local files are turned into `file://` URLs with spaces percent-encoded. `-m` magnifies.

```bash
waveweb http://localhost:3000
waveweb ./build/index.html
waveweb -m "report with spaces.html"
```

### `v` / `e` — quick `wsh` aliases

```zsh
alias v="wsh view"   # v <file/dir>  → open a preview block
alias e="wsh edit"   # e <file>      → open an editor block
```

Use these for one-off blocks without the full `wavep`/`waveweb` path handling: `v notes.md`, `e config.toml`.

## 3. Wave Terminal (`terminal/waveterm/`)

中文：Wave 的全局设置和终端配色，分别由这两个文件管理。

### Settings (`settings.json` → `~/.config/waveterm/settings.json`)

```json
{
  "app:tabbar": "top",
  "autoupdate:channel": "latest",
  "term:fontfamily": "Menlo",
  "term:fontsize": 16,
  "term:theme": "tokyonight",
  "window:opacity": 0.9,
  "window:transparent": true
}
```

| Key | Value | Meaning |
| --- | --- | --- |
| `app:tabbar` | `top` | tab bar across the top of the window |
| `autoupdate:channel` | `latest` | track the latest Wave release |
| `term:fontfamily` | `Menlo` | terminal font |
| `term:fontsize` | `16` | 16 pt |
| `term:theme` | `tokyonight` | use the Tokyo Night palette defined below |
| `window:transparent` + `window:opacity` | `true`, `0.9` | translucent window at 90% opacity |

### Terminal theme (`termthemes.json` → `~/.config/waveterm/termthemes.json`)

This defines the `tokyonight` palette that `term:theme` references — a dark background (`#1a1b27`), light foreground (`#bbc6f6`), and the full 16-color ANSI set plus cursor/selection colors. It is named `"Tokyo Night (from iTerm2)"` because it was derived from the same source palette used for the iTerm2 profile, so the two terminals match.

### Not shipped: `widgets.json` / `wave-status.sh`

中文：源机器上还有一套自定义小组件和一个配额状态脚本，本仓库**故意不带**它们（属于个人化、与本人账号绑定的部分）。

The original Wave setup also had a `widgets.json` (custom sidebar widgets) and a `wave-status.sh` quota status script. These are **intentionally not part of this repo** — they are personal/account-specific. The `wave` function still works without them; the bottom status pane simply stays off unless you supply your own `$WAVE_STATUS_CMD`.

## 4. iTerm2 (`terminal/iterm2/DynamicProfiles/tokyo-night.json`)

中文：iTerm2 用的是同一套 Tokyo Night 配色，做成「动态配置文件（Dynamic Profile）」，这样它和 Wave 看起来完全一致。

iTerm2 supports [Dynamic Profiles](https://iterm2.com/documentation-dynamic-profiles.html): JSON files dropped into `~/Library/Application Support/iTerm2/DynamicProfiles/` that iTerm2 picks up automatically — no manual import. The installer symlinks `tokyo-night.json` there.

The profile is **`Tokyo Night (dotfiles)`** (GUID `dotfiles-tokyo-night-0001`). It was generated from the *same Tokyo Night palette* as the Wave theme, so both terminals look identical. Key settings:

| Setting | Value |
| --- | --- |
| Font (normal + non-ASCII) | `Menlo-Regular 16` |
| Terminal type | `xterm-256color` |
| Window | 120 × 30, transparency `0.1`, blur radius `2` |
| Background | `#1a1b27` (dark navy) |
| Foreground | `#bbc6f6` (lavender) |
| Full ANSI 0–15 | Tokyo Night palette (matches Wave's `termthemes.json`) |

### Set it as your default profile

iTerm2 picks up the file automatically, but you still have to choose it as the default:

1. iTerm2 → **Settings…** → **Profiles**
2. Select **Tokyo Night (dotfiles)** in the list
3. **Other Actions…** (the gear at the bottom) → **Set as Default**

New windows/tabs will now open with the Tokyo Night profile.

### Not shipped: the binary plist

中文：iTerm2 的主配置是个二进制 plist（`com.googlecode.iterm2.plist`），里面混着大量个人设置，**故意不带**。配色用动态配置文件这种干净、可读的方式提供。

iTerm2's main preferences are a binary plist (`com.googlecode.iterm2.plist`) full of machine-specific state. That is **intentionally not shipped**. The Dynamic Profile is the clean, human-readable, version-controllable way to deliver just the theme.

## 5. Fonts

中文：两款字体——MesloLGS Nerd Font 提供 Powerlevel10k 需要的图标字形；Menlo 是终端正文字体。

| Font | Role | How installed |
| --- | --- | --- |
| **MesloLGS Nerd Font** | Powerlevel10k glyphs (powerline arrows, git icons, segment separators) | Homebrew cask `font-meslo-lg-nerd-font`, verified at `~/Library/Fonts/MesloLGSNerdFont-Regular.ttf` |
| **Menlo** | terminal body text in both Wave and iTerm2 | ships with macOS |

The installer adds the Nerd Font in `collect_full_deps` (and it is listed in the `Brewfile`):

```ruby
cask "font-meslo-lg-nerd-font"  # glyphs for Powerlevel10k
```

> If your prompt shows boxes/question marks instead of arrows and icons, your terminal isn't using a Nerd Font. Wave/iTerm2 here use **Menlo** for text, and Powerlevel10k's `instant_prompt`/glyph rendering relies on the **MesloLGS Nerd Font** being installed. If glyphs still look wrong after install, run `p10k configure` and pick the "Yes" option when it asks whether the diamond/lock/etc. icons render correctly.

## Quick reference

| Command | What it does |
| --- | --- |
| `cl` | `clear` |
| `check <term>` | grep your shell history |
| `v <path>` | Wave preview block (`wsh view`) |
| `e <file>` | Wave editor block (`wsh edit`) |
| `wave [dir]` | lay out current Wave tab: shell / claude+codex / file tree |
| `WAVE_AI_PANEL=1 wave .` | same, plus Wave AI side panel |
| `wavep [-m] <path>` | preview a file/dir/url in Wave |
| `waveweb [-m] <file|url>` | open HTML/url in a Wave web block |
| `p10k configure` | regenerate `~/.p10k.zsh` |

Machine-local secrets and overrides go in `~/.zshrc.local` (untracked).
